Context Help

For help with the admin panel, click on the Help icon on any page to learn more about how it works.

Editors

Here's a quick rundown of each of the pages you'll see as an editor:

  • Home - Author dashboard shows you a list of all of your content
  • Incoming Q&A - Lists the questions that have been sent in from readers
  • Content Editor - Go here to post your content.
  • Image Uploader - Go here to upload images (you need to do this before you can put them in an article).
  • Search - Search through content that is already on the site.
  • Old Pages - Lists pages older than 2.5 years
  • Issue Tracker - If something on the site is broken, use this page to report it.
  • Templates - A list of Word templates you can use to send canned responses or start your articles

If you need to change your password, you can do so by clicking the Settings tab.

Also, when you are logged into the admin panel new buttons will appear on the public-facing part of the website. An edit button will appear at the top of each article that you can click on to immediately edit the page you're looking at. This is useful for fixing mistakes or editing old articles you find on the website.

Webmasters

For webmasters, the Home tab is replaced by the webmaster dashboard. It displays a list of content awaiting review (validation of syntax, formatting, etc.).

Some of the pages described above have been moved under the Tools tab. There are two unique admin tools under the Tools tab: the Config Editor which controls the Q&A form, and the Impersonate tool for debugging.

Webmasters also have access to two other tabs:

  • Categories - Add or edit the site's categories
  • User Management - Add new user accounts for the admin panel, and also disable old accounts

At the time of this writing, the category editor is not fully functional so you may need to dig into the database to change some things. Make a database backup before you mess with the database!

Developers

A Subversion repository with source code is maintained by Chris Bednarski, who worked on SexInfo between 2008 and 2009 (and continues to provide some support). If you need an svn account or have questions about the code, send Chris an e-mail.

phpMyAdmin is installed at /sexinfo/admin-sql/; you can find the login information in the configuration file: /core/sex-config.php. It's a good idea to make regular database backups, and to make a backup before changing code or editing the database by hand. Backups are stored in the Google Group (below).

You will need an LSIT account to access the site via FTP to make updates to the code. Contact John to find out how to get an LSIT account. LSIT also manages the server, and if you encounter a problem related to server configuration that you can't resolve you should contact them.

There are various services attached to the sexwebmaster@gmail.com account, including Google Analytics and Google Webmaster Tools. Be sure to monitor these to keep on top of site performance problems and technical issues.

Finally, there is a Google Group for development that contains information on setting up your development environment and suggestions for smooth workflow. You can manage the group (i.e. invite yourself) using the sexwebmaster@gmail.com account. Database backups are also kept here.
